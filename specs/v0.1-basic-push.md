# V0.1: Basic Push

> Builds on: New project.
> Next: [V0.2 — Full Markdown](v0.2-full-markdown.md).

## What's New

User can convert a Markdown file with basic formatting into a Google Doc and get the URL back. Authentication uses a manually-provided token or service account key. Styles are hardcoded defaults that produce a clean, professional-looking document. Rate limiting protects against Google API quota exhaustion from the start.

## Demo

```bash
# Authenticate with a manually-provided token
export GOOGLE_TOKEN="ya29.a0AfH6SM..."

# Push a Markdown file
mdoc push report.md
# Output: https://docs.google.com/document/d/1aBcDe.../edit
```

Given `report.md`:
```markdown
# Q1 Report

## Summary

We shipped **42 features** and fixed _137 bugs_ this quarter.

Visit [our wiki](https://wiki.example.com) for details.

Here is some ~~deprecated~~ text.
```

The resulting Google Doc has:
- "Q1 Report" as a large bold heading
- "Summary" as a smaller bold heading
- Body text in Arial 11pt with bold and italic applied
- A clickable hyperlink
- Strikethrough text rendered correctly

## Capabilities

### Added in this version:
- User can run `mdoc push <file.md>` and get a Google Doc URL
- Paragraphs, headings (H1-H6), bold, italic, strikethrough, links, and line breaks render correctly
- Authentication via environment variable or manually-provided token/service account key
- Hardcoded default styles produce a polished document (Arial, sensible sizes, spacing)
- Proactive rate limiting against Google Docs API quotas with exponential backoff on 429/5xx

### Simplified in this version (improved later):
- Auth is manual (token/service account) — OAuth2 flow via mdoc-api added in V0.5
- Styles are hardcoded — config system added in V0.3
- No frontmatter parsing — added in V0.3
- Creates a new doc every time — upsert (update existing) added in V0.3

### Not yet available:
- Lists, code blocks, inline code, tables, blockquotes (V0.2)
- Config files and style customization (V0.3)
- Frontmatter and upsert (V0.3)
- Images (V0.4)
- Footnotes (V0.4)
- OAuth2 via mdoc-api (V0.5)
- Distribution and auto-update (V1.0)

## Definition of Done

- [ ] `mdoc push file.md` creates a Google Doc and prints the URL to stdout
- [ ] Headings H1-H6 render with distinct, decreasing font sizes and bold styling
- [ ] Bold, italic, and strikethrough text render correctly
- [ ] Links are clickable with correct URLs
- [ ] Line breaks are preserved
- [ ] Paragraphs have appropriate spacing
- [ ] Default styles (Arial font family, sensible sizes/colors/spacing) are applied
- [ ] Authentication works via environment variable or token file
- [ ] Rate limiting prevents API quota exhaustion (proactive RPS limit)
- [ ] Exponential backoff with jitter on 429/5xx responses
- [ ] Exit code 0 on success, non-zero on failure with error message
