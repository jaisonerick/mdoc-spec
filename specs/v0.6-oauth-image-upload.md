# V0.6: OAuth and Image Upload

> Builds on: [V0.5 — mdoc-api](v0.5-mdoc-api.md).
> Next: [V1.0 — Distribution](v1.0-distribution.md).

## What's New

Authentication is now seamless. Users run `mdoc auth`, a browser opens, they log into Google, and tokens are stored automatically. No more manually copying tokens. Multiple Google accounts are supported for users who push docs to different workspaces. Local images in Markdown files are now uploaded via the mdoc-api backend and embedded in the document.

## Demo

```bash
# First-time setup
mdoc auth
# Opens browser → Google login → "Authorization successful!"

# Push with local images
mdoc push report.md
# Output: https://docs.google.com/document/d/1abc.../edit

# Multiple accounts
mdoc auth --account work
mdoc push report.md --account work

# List accounts
mdoc accounts
# Output:
#   default (jaison@gmail.com)
#   work (jaison@company.com)
```

Given `report.md` with a local image:
```markdown
# Performance Report

![CPU Usage](./charts/cpu-usage.png)

CPU usage decreased by 15% after the optimization.
```

The local image `./charts/cpu-usage.png` is uploaded to mdoc-api, which returns a public URL. The image is then embedded in the Google Doc just like URL-based images.

## Capabilities

### Added in this version:
- `mdoc auth` initiates OAuth2 flow via mdoc-api (opens browser, exchanges code, stores token)
- Tokens stored at `~/.config/mdoc/accounts/<name>/token.json`
- Token refresh handled automatically
- `mdoc auth --account <name>` for multiple Google accounts
- `mdoc accounts` lists configured accounts
- `--account` flag on `mdoc push` to select which account to use
- `account` field in frontmatter to set per-document account
- Local image upload via mdoc-api → public URL → `InsertInlineImage`
- Both URL and local file images now work

### Simplified in this version (improved later):
- No image size/scaling control — images inserted at original size
- Tokens stored as plain JSON files — macOS Keychain support is post-V1.0

### Not yet available:
- curl installer and auto-update (V1.0)
- Dry-run mode (post-V1.0)
- Strict mode (post-V1.0)
- Reference commands — `mdoc styles`, `mdoc themes`, `mdoc config`, `mdoc frontmatter` (post-V1.0)
- Table of Contents (post-V1.0)

## Definition of Done

- [ ] `mdoc auth` opens a browser for Google OAuth2 login via mdoc-api
- [ ] Authorization code is exchanged for tokens via mdoc-api (CLI never holds client secrets)
- [ ] Tokens are stored at `~/.config/mdoc/accounts/default/token.json`
- [ ] Expired tokens are refreshed automatically
- [ ] `mdoc auth --account work` stores tokens under the "work" account
- [ ] `mdoc accounts` lists all configured accounts with email addresses
- [ ] `mdoc push file.md --account work` uses the "work" account's tokens
- [ ] Frontmatter `account` field selects the account for that document
- [ ] Local images (relative and absolute paths) are uploaded to mdoc-api
- [ ] mdoc-api returns a public URL for the uploaded image
- [ ] Uploaded images are embedded in the Google Doc via `InsertInlineImage`
- [ ] Missing local image files produce a warning or error (depending on strict mode)
- [ ] Manual token/service account auth from V0.1 continues to work as fallback
