# V1.0: Distribution

> Builds on: [V0.6 — OAuth and Image Upload](v0.6-oauth-image-upload.md).
> Next: Post-V1.0 enhancements.

## What's New

mdoc is now ready for others. Anyone can install it with a single curl command, and it stays up to date automatically. This is the first version you'd share with a teammate or publish publicly.

## Demo

```bash
# Install mdoc
curl -fsSL https://raw.githubusercontent.com/jaisonerick/mdoc/main/install.sh | sh

# Authenticate
mdoc auth

# Push a document
mdoc push report.md
# Output: https://docs.google.com/document/d/1abc.../edit

# Check version
mdoc version
# Output: mdoc v1.0.0 (up to date)

# Later, after a new release:
mdoc push report.md
# stderr: A new version of mdoc is available (v1.0.1). Run 'mdoc update' to upgrade.

mdoc update
# Output: Updated mdoc from v1.0.0 to v1.0.1
```

## Capabilities

### Added in this version:
- curl-based installer script (detects OS and architecture, downloads from GitHub Releases)
- GoReleaser builds for darwin/linux × amd64/arm64
- GitHub Actions release workflow triggered by `v*` tag push
- `mdoc version` shows current version
- Background update check after every command (at most once/day, cached)
- Update notice printed to stderr when a newer version exists
- `mdoc update` downloads and replaces the binary

### Simplified in this version (improved later):
- No Homebrew tap — install via curl only
- Token storage is plain JSON files — Keychain integration is post-V1.0

### Not yet available (post-V1.0):
- `mdoc styles` — show all style properties and defaults
- `mdoc frontmatter` — show all supported frontmatter fields
- `mdoc themes` — list available syntax themes
- `mdoc config` — show resolved merged config
- `--dry-run` — parse and convert without calling Google APIs
- `--strict` — fail on unsupported elements
- `toc: true` — table of contents generation
- Image size/scaling control
- macOS Keychain for token storage
- Homebrew tap

## Definition of Done

- [ ] `curl -fsSL .../install.sh | sh` installs mdoc on macOS (amd64 + arm64) and Linux (amd64 + arm64)
- [ ] Install script detects OS and architecture correctly
- [ ] GoReleaser produces `mdoc_darwin_amd64.tar.gz`, `mdoc_darwin_arm64.tar.gz`, `mdoc_linux_amd64.tar.gz`, `mdoc_linux_arm64.tar.gz`, and `checksums.txt`
- [ ] GitHub Actions workflow builds and publishes a release on `v*` tag push
- [ ] Version is injected via ldflags (`cmd.Version`)
- [ ] `mdoc version` prints the version string
- [ ] Background update check runs after commands (except `update`), at most once per day
- [ ] Update state cached in `~/.config/mdoc/update-state.json`
- [ ] Update notice printed to stderr (never stdout) when a newer version exists
- [ ] `mdoc update` downloads the correct platform binary and replaces the executable
- [ ] Errors during update check are silently ignored
- [ ] All V0.1–V0.5 capabilities work correctly
- [ ] End-to-end: fresh install → auth → push a complex Markdown file → get a correctly formatted Google Doc
