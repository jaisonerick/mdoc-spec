# V0.2: Full Markdown

> Builds on: [V0.1 — Basic Push](v0.1-basic-push.md).
> Next: [V0.3 — Config and Upsert](v0.3-config-upsert.md).

## What's New

Every common Markdown element now renders correctly in the Google Doc. Code blocks get syntax highlighting with configurable themes (default: monokai). Tables use smart column widths. The formatter is now complete — any standard Markdown file produces a professional document.

## Demo

```bash
mdoc push spec.md
# Output: https://docs.google.com/document/d/1xYz.../edit
```

Given `spec.md`:
```markdown
# API Specification

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | /users | List all users |
| POST | /users | Create a user |

### Authentication

All endpoints require a Bearer token:

` ` `python
def authenticate(request):
    token = request.headers.get("Authorization")
    if not token:
        raise Unauthorized("Missing token")
    return verify_jwt(token)
` ` `

Use `Bearer <token>` in the Authorization header.

> **Note:** Tokens expire after 24 hours.

- Step 1: Get a token
- Step 2: Include in headers
  - Sub-step: Use Bearer prefix
- Step 3: Handle expiration

1. First ordered item
2. Second ordered item
```

The resulting Google Doc shows:
- A table with header row (gray background, bold), proper borders, and smart column widths (Method and Path fit to content, Description gets remaining space)
- A syntax-highlighted Python code block in a shaded cell with monokai colors
- Inline code `Bearer <token>` in red monospace
- A blockquote with left border and italic styling
- Nested unordered and ordered lists with proper indentation

## Capabilities

### Added in this version:
- Unordered lists with bullet presets (disc/circle/square)
- Ordered lists with numbered presets (decimal/alpha/roman)
- Nested lists with proper indentation per level
- Fenced code blocks with syntax highlighting via chroma
- Code blocks rendered in 1×1 table cell with themed background
- Adjacent tokens with identical styles merged into single text runs
- Inline code in monospace font at surrounding text size, default red color
- Tables with header row styling (background, bold)
- Smart column width calculation (fit-to-content vs. proportional)
- Table cell borders, padding, and vertical alignment (top)
- Column alignment from Markdown syntax (`:---`, `:---:`, `---:`)
- Blockquotes with left indent and left border
- Syntax theme selection (monokai default, plus dracula, github, solarized-dark, one-dark)

### Simplified in this version (improved later):
- Inline code is always red (no language detection) — improved in V0.3+ when config allows overriding color
- Only one syntax theme active at a time (hardcoded or via future config) — theme selection via config in V0.3
- No config customization for table styles, list spacing, or blockquote appearance — V0.3

### Not yet available:
- Config files and style customization (V0.3)
- Frontmatter parsing and upsert (V0.3)
- Images (V0.4)
- Footnotes (V0.4)
- OAuth2 via mdoc-api and multi-account (V0.5)
- Distribution and auto-update (V1.0)
