# V0.4: Images and Footnotes

> Builds on: [V0.3 — Config and Upsert](v0.3-config-upsert.md).
> Next: [V0.5 — mdoc-api](v0.5-mdoc-api.md).

## What's New

Markdown images that reference URLs are now embedded directly in the Google Doc. Footnotes convert to native Google Docs footnotes. Drive folder placement lets users organize output docs into specific folders.

## Demo

```markdown
---
title: "Architecture Overview"
folder: "Engineering/Docs"
---

# System Architecture

![Architecture Diagram](https://example.com/arch-diagram.png)

The system consists of three main components[^1].

| Component | Role |
|-----------|------|
| API Gateway | Routes requests |
| Worker Pool | Processes jobs |

[^1]: See the detailed design doc for implementation specifics.
```

```bash
mdoc push architecture.md
# Output: https://docs.google.com/document/d/1xyz.../edit
```

The resulting Google Doc:
- Is created in the "Engineering/Docs" folder in Google Drive (auto-created if needed)
- Contains the image from the URL embedded inline
- Has a native Google Docs footnote at the bottom of the page with the reference text

## Capabilities

### Added in this version:
- Images from URLs embedded in the document via `InsertInlineImage`
- Footnotes (`[^1]` syntax) converted to native Google Docs footnotes via `CreateFootnote`
- Drive folder placement via frontmatter `folder` field or config `drive.default_folder`
- Auto-creation of folder path in Google Drive if it doesn't exist

### Simplified in this version (improved later):
- Only URL-based images — local file images require mdoc-api for upload (V0.5)
- No image size/scaling control — images inserted at original size
- No multi-account — uses default/only account (V0.5)

### Not yet available:
- Local file image upload via mdoc-api (V0.5)
- mdoc-api backend service (V0.5)
- OAuth2 via mdoc-api and multi-account (V0.6)
- Distribution and auto-update (V1.0)

## Definition of Done

- [ ] Images referenced by URL are embedded inline in the Google Doc
- [ ] Images with broken/unreachable URLs produce a warning (graceful degradation) or error (strict mode)
- [ ] Local file images produce a warning that they're skipped (upload not yet supported)
- [ ] Footnotes (`[^1]` syntax) render as native Google Docs footnotes
- [ ] Footnote reference markers appear as superscript numbers in the text
- [ ] Footnote content appears at the bottom of the page
- [ ] `folder` in frontmatter creates the doc in the specified Drive folder
- [ ] `drive.default_folder` in config sets the default folder
- [ ] Frontmatter `folder` overrides config `drive.default_folder`
- [ ] Folder paths are auto-created if they don't exist
- [ ] All V0.1–V0.3 capabilities continue to work
